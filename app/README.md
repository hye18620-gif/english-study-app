# 영어 문장 반복 학습 앱 (English Study App)

매일 한국어 10문장을 입력 → 영어로 번역 → **한글 음성 → 2초 후 영어 표시 + 영어 음성**으로
반복 학습하는 **모바일 PWA**입니다. Claude Design 프로토타입(`project/English Study App.dc.html`)을
실제 모바일에서 구동되는 앱으로 구현했습니다.

설치형 PWA라 홈 화면에 추가하면 일반 앱처럼 전체 화면으로 실행되고, 한 번 열어두면
오프라인에서도 동작합니다. (자동 번역만 인터넷이 필요)

## 기능 (요구사항 1–11 매핑)

| # | 요구사항 | 구현 |
|---|----------|------|
| 1 | 한국어 10문장 → 영어 번역 | 문장 입력 → Claude(Anthropic) API 자동 번역, 키 없으면 영어 직접 입력 |
| 2 | 전체 / 한 문장씩 선택 반복 | 학습 화면 상단 번호 점(1–10)으로 문장 선택, 듣기로 현재 문장 반복 |
| 3 | 1번부터 차례로 표시 | 순서대로 진행, ‹ › 로 이동 |
| 4 | 한글 먼저 + 한글 음성 → 2초 후 영어 표시 + 영어 음성 | 듣기 버튼: 한글 TTS → 2초 → 영어 표시 + 영어 TTS (기기 음성 사용) |
| 5 | 다음날 전일/미완료 문장 먼저 복습 | 홈 상단 "미완료 문장" 알림 + 복습하기 |
| 6 | 문장마다 완료 버튼 | 학습 화면 "완료 표시하기" 토글, 복습 여부 결정 |
| 7 | 토요일 = 주간 전체 복습 | 토요일 자동으로 주간 복습 카드 표시 |
| 8 | 매월 마지막 일요일 = 월간 미완료 복습 | 해당일 자동으로 월간 복습 카드 표시 |
| 9 | 일자 선택 시 해당일 전체 연습 | 달력 탭에서 날짜 탭 → 그날 문장 학습 |
| 10 | 학습 내용 그래픽 확인 | 통계 탭: 7/30일 완료율 차트, 연속 학습일, 학습 기록 |
| 11 | 모바일 최적화 | 430px 폭, 터치 UI, 안전영역(safe-area) 대응, 설치형 PWA |

## 실행 방법

순수 정적 파일이라 **빌드가 필요 없습니다.** 정적 서버로 `app/` 폴더를 서빙하면 됩니다.
(서비스 워커/PWA 설치는 `https://` 또는 `localhost`에서만 동작합니다.)

### 로컬에서 빠르게 보기

```bash
cd app
python3 -m http.server 8000     # 또는: npx serve .
```

브라우저에서 `http://localhost:8000` 접속.

### 실제 휴대폰에서 사용 (홈 화면에 설치)

1. `app/` 폴더를 HTTPS 정적 호스팅에 올립니다.
   예: GitHub Pages, Netlify, Vercel, Cloudflare Pages 등 (어디든 정적 배포 가능).
2. 휴대폰 브라우저로 그 주소를 엽니다.
3. **설치**
   - **iOS Safari**: 공유 → "홈 화면에 추가"
   - **Android Chrome**: 메뉴(⋮) → "앱 설치" / "홈 화면에 추가"
4. 홈 화면 아이콘으로 실행하면 전체 화면 앱처럼 동작합니다.

> GitHub Pages 예시: 저장소 Settings → Pages → 브랜치를 선택하고
> 폴더를 `/app`(또는 `app/`를 루트로 배포)으로 지정하면 `https://<사용자>.github.io/<repo>/`에서 열립니다.

## 자동 번역 설정 (선택)

프로토타입은 디자인 도구 내부의 `window.claude.complete`로 번역했지만, 실제 앱에는 그것이 없습니다.
대신 **본인의 Anthropic API 키**로 브라우저에서 직접 번역합니다.

- 우측 상단 **⚙️ 설정** → API 키(`sk-ant-...`) 입력 → 저장.
- 키는 **이 기기(브라우저 localStorage)에만** 저장되며 별도 서버로 전송되지 않습니다.
- 키 입력은 선택사항입니다. 키가 없으면 번역 결과 화면에서 **영어를 직접 입력**해 그대로 학습할 수 있습니다.
- 사용 모델: `claude-haiku-4-5`.

보안 메모: 프런트엔드 전용 앱이라 키가 기기에 저장됩니다. 개인 학습용으로는 충분하지만,
여러 사용자에게 배포하거나 키 노출이 걱정되면 작은 백엔드(서버리스 함수)로 번역 요청을
프록시하도록 바꾸는 것을 권장합니다. (현재 구조에서 `translate()`의 `fetch` 대상만 바꾸면 됩니다.)

## 데이터 저장

- 학습 데이터는 브라우저 `localStorage` 키 `eng_learn_v1`에 저장됩니다(기기 로컬).
- 앱/브라우저 데이터를 지우면 학습 기록도 사라집니다.

## 음성(TTS) 참고

- 브라우저 내장 **Web Speech API**(한국어 `ko-KR`, 영어 `en-US`)를 사용합니다.
- 모바일에서는 음성 재생이 사용자 탭(듣기 버튼) 이후에만 시작됩니다(브라우저 정책).
- 기기에 해당 언어 음성이 없으면 소리 없이 다음 단계로 진행됩니다.

## 파일 구조

```
app/
├── index.html              # 앱 셸 + 폰트/매니페스트/서비스워커 등록
├── styles.css              # 전역 리셋 + 애니메이션 (대부분의 스타일은 app.js 내 인라인)
├── app.js                  # 전체 앱 로직/렌더링 (프로토타입 Component 클래스 포팅)
├── manifest.webmanifest    # PWA 매니페스트
├── service-worker.js       # 오프라인 캐시(앱 셸)
└── icons/                  # 앱 아이콘 (PWA / iOS 홈 화면)
```

아이콘은 `scripts/generate_icons.py`(파이썬 표준 라이브러리만 사용)로 생성합니다:
`python3 scripts/generate_icons.py`.
